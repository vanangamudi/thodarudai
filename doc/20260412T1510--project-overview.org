* Project: Tamil Splits Toolkit and GUI

This repository contains a set of tools and a Qt GUI to explore Tamil word lists, query by prefix/suffix/regex, curate and commit split annotations, build on-disk tries for fast lookup, export datasets, and train a simple tokenizer model. It also includes a small phonetic input/transliteration module and demo.

This document introduces each module, its public classes, functions, variables, and their purposes. Identifiers are shown as =verbatim=; inline code and command examples use ~code~.

- Audience: developers and contributors completely new to this codebase.
- Conventions:
  - Paths and file names are literal.
  - Use ~python -m package.module ...~ to run modules where applicable.

Contents:
- Data and generated files
- Python modules and packages
- Command-line entry points
- Profiles and directory layout
- Notes and org docs

  - Format: org-mode table per file section with columns: length, start-end lines, and fully qualified function name.

  - Word list with metadata. Header: "word<TAB>freq<TAB>glen".
  - Created by your own pipeline or via ~tools/build_word_index.py~ (aggregates raw sources).

  - Ledger log of commits from GUI. Header: "timestamp<TAB>batch<TAB>id<TAB>word<TAB>splits<TAB>notes".
  - Includes special =__SUMMARY__= entries with JSON snapshot of curation progress.

- =data/ui-log.tsv=
  - Lightweight UI event log (timestamp, event_type, parameters) for analytics.

- =data/reminders.tsv=
  - Bag of words to revisit in GUI. Header: "timestamp<TAB>word<TAB>notes".

* Module: =tools/word_indexer.py=

Purpose: in-memory word index with multiple indices (alpha, reverse, length) for fast prefix/suffix queries, filtering, and canonical ranking.

- Class =WordIndex=
  - =__init__(wordlist_path)=
    - Loads word list TSV, builds indices, and prepares regex cache.
  - Attributes:
    - =wordlist_path=: source TSV path.
    - =words=: list of tuples =(word, freq, glen)= sorted by canonical key (-glen, -freq, word).
    - =by_word=: list of =(word, freq, glen)= sorted alphabetically by =word=.
    - =_words_only=: list of =word= strings aligned with =by_word= for bisect.
    - =by_rev=: list of =(rev_word, word, freq, glen)= sorted by =rev_word=.
    - =_revs_only=: list of reversed strings aligned with =by_rev= for bisect.
    - =len_ranges=: dict glen -> (start_idx, end_exclusive) into =words=.
    - =len_counts=: dict glen -> count of words with that length.
    - =order_index=: dict word -> index position in =words= order.
    - =glen_map=: dict word -> glen.
    - =index_words=: set of all words for quick membership.
    - =_regex_cache=: (reserved) mapping pattern -> compiled regex.
  - Methods (internal helpers):
    - =_compile_regex(pattern)=: returns compiled regex or None if invalid/empty.
    - =_select_candidates(prefix, suffix, min_len, max_len)=: chooses best base candidate list based on filters.
    - =_passes_filters(rec, prefix, suffix, compiled_rx, min_len, max_len, exclude_fn)=: evaluates a single record against all filters.
    - =_filter_candidates(candidates, ...)=: filters a list using =_passes_filters=.
    - =_finalize_results(filtered, offset, limit)=: sort canonically by (-glen, -freq, word), apply offset/limit.
    - =_load_wordlist()=: parses TSV file into =words=, sorts canonically.
    - =_build_indices()=: builds all secondary indices and maps.
    - =_prefix_bounds(prefix)=: returns (lo, hi) in =_words_only= for words starting with =prefix=.
    - =_suffix_bounds(suffix)=: returns (lo, hi) in =_revs_only= for reversed-suffix range.
    - =_rank_key(rec)=: (-glen, -freq, word).
    - =_candidates_from_prefix(prefix, min_len, max_len)=: range-scan on =by_word= and apply length prefilter.
    - =_candidates_from_suffix(suffix, min_len, max_len)=: range-scan on =by_rev= and apply length prefilter.
    - =_candidates_from_lengths(min_len, max_len)=: iterate =len_ranges= descending and extend slices.
  - Methods (API):
    - =query_words(prefix="", suffix="", min_len=1, max_len=None, limit=200, offset=0, exclude_fn=None, regex="")=:
      - Returns sorted list of =(word, freq, glen)= that match filters.
      - Efficient candidate selection based on provided filters.

    - =mm=: memory map object.
    - =flush_per_write=: bool controlling per-write flush.
  - Methods:
    - =close()=: flush and close mmap/file.
    - =allocate_node(node_data: bytes) -> int=:
      - Appends a record to end, writes =node_data=, returns file offset.
      - Note: free list not yet used (always appends).
    - =read_node(offset: int) -> bytes=:
      - Reads node by offset using record length stored in first 4 bytes.
    - =write_node(offset: int, node_data: bytes) -> int=:
      - Overwrites in place if size fits current record; else appends and returns new offset.
      - Caller must update parent pointer when reallocation occurs.


* Module: =lib/chorkilai/mlm/chorkilai/trie.py=

Purpose: both in-memory and on-disk trie implementations for Tamil grapheme sequence storage and lookup.

- Constants:
  - =XSV_DELIMITER=: '\t' for TSV parsing.
  - =LETTER_FIELD_SIZE=: 8, max bytes to store a token key per child record.

- Abstract Base: =BaseTrie=
  - =add(item)=: insert token sequence or word.
  - =lookup(word: str) -> bool=: exact match lookup.
  - =prefix_exists_p(prefix: str) -> bool=: prefix existence check.
  - =close()=: no-op by default (overridden by on-disk implementation).

- Helper class: =Node=
  - Attributes:
    - =value=: token or character stored (for in-memory representation).
    - =count=: frequency/count placeholder (unused in code paths).
    - =children=: dict token -> child Node.
    - =is_complete=: True if path is terminal (word).
    - =level=: depth for debug printing.
  - Methods:
    - =__repr__/__str__()=: pretty-print subtree up to certain depth.

- In-memory =Trie(BaseTrie)=
  - =__init__()=: create root =Node=.
  - Internal helpers for on-disk variant interop (in this file some remnants exist).
  - =add(item)=: insert a sequence of tokens or characters into child dicts; set =is_complete= at terminal.
  - =lookup(word: str) -> bool=: traverse by characters, return =is_complete=.
  - =find_prefix(prefix, default=None)=: finds node for given prefix; returns (node, last_index_matched).
  - =prefix_exists_p(prefix: str)=: True if traversal completes without missing node.
  - =get_all_suffixes(prefix)=:
    - Collects suffix strings under node matching =prefix=.
  - =words()=: returns suffix set from root.

- On-disk =OnDiskTrie(BaseTrie)=
  - Constants:
    - =HEADER_SIZE=: 9 (record header inside node entries).
    - =INITIAL_NODE_CAPACITY=: default children slots per node (resizable).
    - =CHILD_RECORD_SIZE=: computed size per child record.
  - =__init__(db_path, new=False)=:
    - Opens =TrieDiskStore= and initializes root node or reads existing header for root offset.
    - Attributes:
      - =store=: =TrieDiskStore= instance.
      - =root_offset=: file offset of root node.
      - =_root_offset=: duplicate for find/traversal.
  - Internal serialization/storage:
    - =_node_record_size(capacity) -> int=.
    - =_read_node(offset) -> dict=:
      - Returns dict with keys: offset, record_length, is_terminal, child_count, capacity, children=[{key, child_ptr}, ...].
    - =_serialize_node(node_dict) -> bytes=:
      - Produces fixed-size buffer per capacity; trims/pads =key= bytes to =LETTER_FIELD_SIZE=.
    - =_create_empty_node() -> int=:
      - Allocates a blank node, returns new offset.
    - =_expand_node(parent_offset, parent_node, child_key, current_offset, node) -> int=:
      - Doubles capacity, writes node; updates parent pointer if offset moved; updates header if root moved.
    - =_update_header()=:
      - Writes store header (magic "TRIE", version=1, free_list_head=0, root_offset).
  - API:
    - =add(item)=:
      - Iterates token sequence; expands nodes as needed; creates children; marks final node as terminal.
    - =lookup(word: str) -> bool=:
      - Traverses characters; returns terminal flag of final node or False if any edge missing.
    - =prefix_exists_p(prefix: str) -> bool=:
      - Like lookup but without requiring terminal at end.
    - =find_prefix(prefix, default=None)=:
      - Traverses using internal =_read_node= from =_root_offset=; returns node dict or default.
    - =get_all_suffixes(prefix)=:
      - Uses recursive helper =_collect_suffixes(node, current_suffix)= to accumulate all suffix strings.
    - =close()=: closes underlying =TrieDiskStore=.

- Utility functions:
  - =get_trie(use_mmap=False, db_path=None, db_flush=False)=:
    - Returns =OnDiskTrie= when =use_mmap=True= else in-memory =Trie=.
  - =load_files(filepaths, trie, pbarp=False, max_entries=None)=:
    - Loads TSV word files; uses ~arichuvadi.get_letters_coding(token)~ to add sequences to trie.
  - =build_trie(filepaths, use_mmap=False, db_path=None, db_flush=False, pbarp=True, max_entries=None)=:
    - Creates a trie and optionally loads files if =db_flush=True= (fresh build).

- CLI (when run as script):
  - =--build= / =--query=: build or query modes.
  - =--mmap=, =--mmap-flush=, =--db-path=: control on-disk trie.
  - =--lookup= exact word query; =--prefix= prefix collection; =--limit= result cap; =--pbar= show tqdm.
  - In query mode without mmap: requires file paths to load into in-memory trie.

* Module: =tools/tamil_phonetic.py=

Purpose: standalone transliteration logic for Tamil phonetic input using greedy token matching.

- Constants:
  - =PHONETIC_VOWELS=: map roman vowel tokens to standalone Tamil vowels.
  - =VOWEL_DIACRITICS=: combining marks applied to consonant when vowel follows.
  - =CONSONANTS=: map roman consonant tokens to Tamil base consonants.
  - =PULLI=: "்" virama mark to suppress inherent vowel.
  - =VOWEL_TOKENS=: sorted vowel tokens by length desc for greedy match.
  - =CONSONANT_TOKENS=: sorted consonant tokens by length desc.

- Function:
  - =transliterate(text: str) -> str=
    - Greedy scan over input:
      - When vowel token matches, attach to pending consonant (or emit standalone).
      - When consonant token matches, hold as pending; flush prior pending with pulli if unresolved.
      - On whitespace/punctuation: flush with pulli if needed, then emit char.
      - On other chars: flush pending raw, then emit char.
    - Ensures pending consonant at end is closed with a virama pulli.

- CLI demo (when run as script): prints transliteration of first argument.

Note: =tamilphonetic/core.py= contains the same logic as a package-friendly module (see below).

* Module: =tools/train_tokenizer.py=

Purpose: simple sequence-to-sequence model to map surface words to tokenized splits (e.g., Tamil grapheme tokens) using PyTorch.

- Special tokens:
  - =PAD= "<pad>", =BOS= "<bos>", =EOS= "<eos>"

- Data I/O:
  - =load_dataset(path) -> List[Tuple[str,str]]=:
    - Reads TSV with header "src<TAB>tgt"; returns list of (src, tgt) pairs.
  - =build_vocabs(pairs, token_delim="-") -> (src2i, i2src, tgt2i, i2tgt)=:
    - Builds character-level source vocab and target token vocab (split by delimiter).
  - Class =TokDataset(Dataset)=:
    - =__init__(pairs, src2i, tgt2i, token_delim="-", max_src=None, max_tgt=None)=
    - =__len__()=
    - =__getitem__(idx) -> (src_ids, tgt_ids)=: adds BOS/EOS and maps to indices.
  - =collate(batch)=:
    - Pads sequences to batch max length; returns tensors: src, src_lens, tgt, tgt_lens.

- Model:
  - Class =Encoder(nn.Module)=:
    - LSTM-based encoder with embeddings; =forward(x, lengths)= returns final (h, c).
  - Class =Decoder(nn.Module)=:
    - LSTM-based decoder with embeddings and linear projection; =forward(y_in, h, c)= returns logits, (h, c).
  - Class =Seq2Seq(nn.Module)=:
    - Connects encoder and decoder; =forward(src, src_lens, tgt, teacher_forcing=0.5)= returns logits sequence.

  - ~run_infer(words, model, cfg, src2i, tgt2i, i2tgt, device="cpu", max_len=None, batch_size=256)~: returns list of (word, pred_tgt).
  - ~evaluate(dataset_path, model, cfg, src2i, tgt2i, i2tgt, device="cpu", max_len=None)~: prints exact-match accuracy.
  - ~load_model(model_dir, device="cpu") -> (model, cfg, src2i, tgt2i, i2tgt)~

  - =maybe_run_eval_and_infer(args, model, cfg, src2i, i2tgt, tgt2i)=
