* Project Documentation

This document describes every module, class, function, and module-level variable in the repository, calls out stale or dead code, and highlights inconsistencies and potential bugs. Identifiers are shown as =verbatim= and short code/commands are shown with ~inline code~.

** Repository layout
- =tools/gui_qt_app.py=: Qt GUI client for querying words, editing splits, sorting, and committing edits either locally or to a Unix socket server.
- =tools/word_server.py=: Unix domain socket word server that supports =QUERY=, =COMMIT=, =RELOAD=, and =STATS= commands.
- =tools/word_indexer.py=: Loads a word index file (=word-index.tsv=) and supports query filters.
- =tools/build_word_index.py=: Aggregates frequency counts from input files and prints a new TSV with grapheme lengths; utility script.
- =tools/tamil_phonetic.py=: Standalone Tamil transliteration logic (romanized to Tamil), used by the GUI for phonetic input fields.
- =tamilphonetic/core.py=: Library form of the transliteration logic (duplicate of =tools/tamil_phonetic.py= with minor differences).
- =tamilphonetic/__init__.py=: Exports the transliteration API from the =tamilphonetic= package.
- =tamil_editor_demo.py=: Simple QTextEdit-based demo of phonetic input; not integrated with the main GUI.
- =emacs/tamil-splits.el=: Emacs client for issuing queries and committing TSV batches to the server.
- =trie.py=: A generic Trie implementation geared to arbitrary sequence types; includes =StringNode= specialization.
- =VARALARU.org=: Log/instructions for external tools and data processing.
- =VAZHU.org=: Notes/log including a demonstration of a trie bug and its fix.

** Data and runtime
- Profiles (=tools/profile.py=) define per-profile paths under =<base_dir>/<profile>/=:
  - =word-index.tsv=: Word index file with header =word	freq	glen=.
  - =ledger.tsv=: Ledger file written by GUI-in-local-mode or by the server.
  - =batches/=: Batch TSVs in local-commit mode.
  - =tamil-words.sock=: Unix domain socket path for the server.
- GUI default mode is ="server"= but queries are performed locally via =WordIndex=; commits can go local or server depending on mode.

* Module: tools/tamil_phonetic.py

- Module purpose: Transliterate romanized Tamil sequences into Tamil script with greedy token matching; add pulli marker by default for bare consonants.
- Module-level constants:
  - =PHONETIC_VOWELS=: dict mapping vowel tokens (='a', 'aa', ..., 'au'=) to standalone Tamil letters.
  - =VOWEL_DIACRITICS=: dict mapping vowel tokens to diacritics used with a pending consonant (='' for ='a'=).
  - =CONSONANTS=: dict mapping consonant tokens to Tamil base consonants. Notes:
    - Case-sensitive tokens exist (='N', 'L', 'S', 'Z', 'ksH'=), with special handling in transliteration.
    - =ksh= maps to a ligature form with ZWNJ (=க்‌ஷ=).
  - =PULLI=: Tamil virama marker "்".
  - =VOWEL_TOKENS=: list of vowel tokens sorted by descending length (for greedy match).
  - =CONSONANT_TOKENS=: list of consonant tokens sorted by descending length (for greedy match).
- =def transliterate(text)=
  - Purpose: Convert romanized =text= to Tamil, greedily matching tokens and applying diacritics to pending consonants; append =PULLI= for bare consonants at boundaries/end.
  - Behavior:
    - Maintains a =pending= consonant. When a vowel token is found:
      - If =pending= exists, appends =pending + VOWEL_DIACRITICS[vowel]=.
      - Else appends standalone vowel letter =PHONETIC_VOWELS[vowel]=.
    - For consonant tokens:
      - Flushes any previous =pending= with =PULLI=, then sets =pending= to the matched consonant letter.
      - Accepts exact match or case-insensitive match only when both the input candidate’s and token’s first letters are lowercase (preserving explicit capital-only tokens behavior).
    - On whitespace/punctuation boundary:
      - Flushes =pending= with =PULLI=.
    - On other unmatched characters:
      - Flushes =pending= without pulli (pure consonant), then copies the character.
    - At end of input:
      - Flushes =pending= with =PULLI=.
  - Returns: A single Tamil string.
- CLI guard:
  - If invoked as a script, prints transliteration for the provided argument.
- Notes:
  - Behavior around case-insensitive consonants is asymmetric to allow explicit capitalized tokens like ='N', 'L', 'S', 'Z'= to remain distinct.

* Module: tools/word_indexer.py

- Module purpose: Load a word index and provide filtered queries.
- =class WordIndex=
  - =__init__(self, wordlist_path)=
    - Purpose: Construct index bound to a TSV file.
    - Parameters: =wordlist_path= path to a TSV with header =word	freq	glen=.
    - Attributes:
      - =self.wordlist_path=: stored path.
      - =self.words=: list of tuples =(word, freq, glen)= sorted by =(-glen, -freq, word)=.
    - Calls: =self._load_wordlist()=.
  - =def _load_wordlist(self)=
    - Purpose: Read and parse the TSV file; skip blank or bad rows.
    - Behavior:
      - Reads header; builds column index by name.
      - Appends parsed tuples.
      - Sorts descending by grapheme length, then frequency, then ascending word.
    - Side-effects: Prints "loading word list..." start and DONE messages to stdout.
  - =def query_words(self, prefix="", suffix="", min_len=1, max_len=None, limit=200, offset=0, exclude_fn=None, regex="")=
    - Purpose: Return words that satisfy various filters.
    - Parameters:
      - =prefix=: required =str.startswith(prefix)= filter.
      - =suffix=: required =str.endswith(suffix)= filter.
      - =min_len=: minimum =glen= inclusive.
      - =max_len=: optional maximum =glen= inclusive; =None= to ignore.
      - =limit=: maximum result count.
      - =offset=: number of matches to skip before collecting.
      - =exclude_fn=: callable(word) -> bool to skip accepted/other words.
      - =regex=: optional regex pattern; compiled if valid; =search()= used.
    - Returns: list of tuples =(word, freq, glen)= at most =limit= items.

* Module: trie.py

- =class Node=
  - =__slots__=: =['count','is_complete','children','trie']=
  - =__init__(self, trie, count=0)=

      - Resolving =(prefix_node, length)= via =find_prefix=.

      - Else: returns =[prefix[:length] + suffix for suffix in prefix_node._get_suffixes()]=.

* Appendix: Data formats

- Word index TSV (=word-index.tsv=):
  - Header: =word	freq	glen=
  - Rows: Tamil word text, integer frequency, integer grapheme length.
- GUI-local batch TSV (current):
  - Header: =id	split	freq	glen	status	notes=
  - Rows: id string, split text (word), frequency, grapheme length, status, notes.
- Emacs/server-required batch TSV:
  - Header: =id	word	freq	glen	splits	status	notes=
- Server ledger (=LEDGER_HDR=):
  - Header: =timestamp	batch	id	word	status	splits	notes=
- GUI-local ledger (=update_ledger=):
  - Header: =timestamp	batch	id	split	status	notes= (conflicts with server’s expected ledger header).
